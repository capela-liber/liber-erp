Metabooks Integration

  • Metabooks Setting -> General Settings panel:

    Fill the form with Metabooks Username, password and provided code.

  • Metabooks Sync Processing Status inside Odoo

    Sync every one(1) day the metabooks data and creates a log inside Sales with the processing status.

  • Updating only one product metabooks

    This button allows the user to update only one book data instead of the hole metabooks database.

  • Info from metabooks inside product

    Data that has been pulled from metabooks.

Testes

  • Teste automático (sem credencial, sem rede)

    A camada HTTP é mockada num único ponto: metabooks.connector._get_client é substituído por um cliente falso, então nenhuma credencial de ir.config_parameter é lida e nenhuma chamada de rede acontece. Verifica-se o parsing + upsert: dado um payload conhecido da API, os campos metabooks_* corretos são gravados no produto (import por ISBN, idempotência, "não encontrado" e import de catálogo de editora). Rodar: 

    odoo -d copyright19 -u liber_metabooks_integration \
  --test-enable --test-tags '/liber_metabooks_integration:TestMetabooksSync' \
  --http-port=8072 --stop-after-init
  • Smoke test manual (sync real de uma editora)

    Para exercitar a integração de ponta a ponta contra a API real — por exemplo o catálogo da editora BR0089701 (um id VL / mvbId, não um ISBN):

    1. Em Definições → Metabooks, preencha usuário e senha reais. As credenciais ficam em ir.config_parameter e nunca no código nem no repositório.
    2. Use o assistente "Importar por editora" e informe BR0089701, ou dispare um metabooks.import.job para o catálogo completo (roda em segundo plano, retomável).
    3. Confira que os produtos foram criados/atualizados com título, editora, autores e dados técnicos, e que a capa baixou.

    Este passo depende de credencial válida e de rede, por isso fica fora do teste automático — é uma verificação humana, sob demanda.
